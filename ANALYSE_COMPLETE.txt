✅ ARCHITECTURE ACTUELLE
   ├─ Monorepo pnpm avec 3 packages backend
   ├─ Frontend React 18 + Tailwind CSS + Material-UI
   ├─ Langium DSL pour contraintes planning
   ├─ Express API stateless (GET /api/health, POST /api/solve)
   └─ MiniZinc solver external (Highs backend)

✅ MODELES DE DONNEES
   ├─ PlanningData interface (frontend/src/model/Planning.ts)
   ├─ Langium AST auto-généré (backend/packages/language/src/generated/)
   ├─ Types: Time, Activities, Resources, Roles, Constraints, Preferences
   └─ Actuellement: Pas de persistance (volatile, in-memory)

✅ CYCLE DE VIE PLANNING
   ├─ CREE: 6-step wizard → JSON payload
   ├─ VALIDE: Langium parser + validator
   ├─ GENERE: MiniZinc code generation (441 lignes)
   ├─ RESOLU: Minizinc solver execution
   └─ PERDU: Cleanup des temp files (❌ problème : no history)

✅ PERSISTANCE BASE
   ├─ Schema PostgreSQL proposé (7 tables)
   ├─ users (id, email, password_hash)
   ├─ plannings (id, user_id, data JSONB, created_at)
   ├─ planning_versions (id, planning_id, version, snapshot)
   ├─ planning_solutions (id, planning_id, minizinc_output)
   ├─ planning_shares (id, planning_id, shared_with_user_id)
   ├─ audit_logs (action, actor_id, target_id, timestamp)
   └─ Indexes, foreign keys, soft deletes inclus

✅ AUTHENTIFICATION EMAIL/PASSWORD
   ├─ Register: POST /auth/register (email, password)
   │  └─ Hash with bcryptjs, store user, return JWT
   ├─ Login: POST /auth/login (email, password)
   │  └─ Verify hash, return JWT token (exp: 7 days)
   ├─ Middleware: authMiddleware(token) → req.user = {id, email}
   ├─ Protected routes: /api/planning/* (check req.user.id)
   └─ Frontend: AuthContext + ProtectedRoute wrapper

✅ RISQUES IDENTIFIES
   1. Password security (mitigation: bcryptjs + salt rounds 12)
   2. JWT token expiration (mitigation: refresh tokens in Phase 2.2)
   3. Database connection loss (mitigation: retry logic + connection pool)
   4. Concurrent modifications (mitigation: optimistic locking)
   5. Migration backward compatibility (mitigation: schema versioning)
   6. XSS attacks (mitigation: React auto-escape + CSP headers)
   7. SQL injection (mitigation: parameterized queries + ORM validation)
   8. Timezone issues (mitigation: UTC timestamps + ISO 8601)
   9. Large planning data (mitigation: JSONB compression)
   10. Lost connections during solve (mitigation: async queue + retry)
